# telemt-stats

Консольный инструмент для мониторинга Telegram MTProto прокси (telemt).
Запрашивает Prometheus-метрики и выводит читаемый отчёт на русском языке.

## Требования

- **bash**
- **curl** — загрузка метрик
- **awk** — парсинг и форматирование
- Запущенный telemt-прокси с включёнными Prometheus-метриками

## Установка

```bash
sudo apt install curl awk   # Debian/Ubuntu
# или
sudo dnf install curl awk   # RHEL/Fedora
```

```bash
git clone https://github.com/sh-222/telemt-stats.git
cd telemt-stats
chmod +x stat
```

## Настройка

По умолчанию скрипт обращается к `http://127.0.0.1:9090/metrics`.

Если прокси запущен в Docker, на другом порту или на удалённом сервере —
задайте переменную окружения `TELEMT_METRICS_URL`:

```bash
export TELEMT_METRICS_URL="http://192.168.1.10:9090/metrics"
./stat
```

Типовые сценарии:

| Сценарий | URL |
|----------|-----|
| Прокси локально, порт 9090 | `http://127.0.0.1:9090/metrics` |
| Прокси в Docker (проброс `-p 9090:9090`) | `http://127.0.0.1:9090/metrics` |
| Прокси в Docker (`--network host`) | `http://127.0.0.1:9090/metrics` |
| Удалённый сервер | `http://<ip>:9090/metrics` |
| Нестандартный порт | `http://127.0.0.1:<port>/metrics` |

## Использование

```bash
./stat
```

Для удобства можно добавить алиас в `~/.bashrc` или `~/.zshrc`:

```bash
alias telemt-stats='TELEMT_METRICS_URL="http://..." /path/to/stat'
```

## Устранение неполадок

- **Ошибка: не удалось получить данные** — проверьте, что прокси запущен и
  метрики доступны: `curl $TELEMT_METRICS_URL`. Если прокси в Docker —
  опубликуйте порт 9090 (`-p 9090:9090`) или используйте `--network host`.
- **Неверные/нулевые значения** — убедитесь, что прокси собирает метрики
  (включите Prometheus-экспорт в конфиге telemt).
